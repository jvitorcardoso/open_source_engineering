# Guia de Contribuição

Obrigado por considerar contribuir para o Open Source Engineering! Este documento fornece diretrizes para contribuições efetivas.

## 🎯 Como Contribuir

### Reportando Issues

- Use a busca para verificar se o issue já não foi reportado
- Forneça uma descrição clara e detalhada do problema
- Inclua exemplos de código quando relevante
- Especifique versões de software e ambiente

### Sugerindo Melhorias

- Abra um issue descrevendo a melhoria proposta
- Explique por que a melhoria seria útil
- Forneça exemplos de uso quando possível

### Pull Requests

1. **Fork** o repositório
2. **Clone** seu fork localmente
3. **Crie** uma branch para sua feature
4. **Implemente** suas mudanças
5. **Teste** suas mudanças
6. **Commit** com mensagens descritivas
7. **Push** para sua branch
8. **Abra** um Pull Request

### Padrões de Código

- Siga as convenções de nomenclatura estabelecidas
- Mantenha o código limpo e bem documentado
- Inclua comentários explicativos quando necessário
- Siga as práticas recomendadas para cada tecnologia

### Estrutura de Commits

Use mensagens de commit claras e descritivas:

```
tipo(escopo): descrição breve

Descrição mais detalhada se necessário.

- Lista de mudanças específicas
- Outras mudanças relevantes
```

Tipos recomendados:
- `feat`: nova funcionalidade
- `fix`: correção de bug
- `docs`: mudanças na documentação
- `style`: formatação, código que não afeta funcionalidade
- `refactor`: refatoração de código
- `test`: adição ou correção de testes
- `chore`: tarefas de manutenção

## 🧪 Testes

- Inclua testes para novas funcionalidades
- Certifique-se de que todos os testes passam
- Mantenha a cobertura de testes alta

## 📚 Documentação

- Atualize a documentação para mudanças relevantes
- Inclua exemplos de uso quando apropriado
- Mantenha o README.md atualizado

## 🤔 Dúvidas?

Se você tiver dúvidas sobre como contribuir, abra um issue com a tag `question` ou entre em contato através do GitHub.

Agradecemos suas contribuições! 🎉
